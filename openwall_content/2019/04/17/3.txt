When verifying HTTPS connections when an SSLContext is passed to urllib3, system CA certificates will be loaded into the SSLContext by default in addition to any manually-specified CA certificates. This causes TLS handshakes that should fail given only the manually specified certs to succeed based on system CA certs.  This affects urllib3 and below. The fix has been released in version  The vulnerability was reported by Christian Heimes.  A CVE ID has been requested, will follow up with it when we have it.
