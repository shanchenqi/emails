SubProject: All   Severity: Moderate   Versions: and all previous 1.5 releases   Exploit type: Session fixation   Reported Date:   Fixed Date:  Description  Session id doesn't get modified when user logs in.  A remote site may be   able to forward a visitor to the Joomla! site and set a specific cookie.    If the user then logs in, the remote site can use that cookie to   authenticate as that user.
