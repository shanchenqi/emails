Insufficient path sanitization on several query string parameters leads to inclusion of arbitrary files from remote sources, this could be exploited to execute arbitrary command or code.  The vulnerable URLs are similar to the one referenced in a previously disclosed file inclusion vulnerability affecting the same version of the software (see References). It has been discovered that 'body', 'footer', 'header', 'menu_left', 'menu_right' are also vulnerable to remote file inclusion.
