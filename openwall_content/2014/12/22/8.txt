The unzip command line tool is affected by heap-based buffer overflows within the CRC32 verification, the test_compr_eb() and the getZip64Data() functions. The input errors may result in in arbitrary code execution.  A specially crafted zip file, passed to unzip -t, can be used to trigger the vulnerability.  Affected version:  UnZip <= 6.0  Fixed version:  UnZip, N/A  Credit: vulnerability report received from the Google Security Team.
