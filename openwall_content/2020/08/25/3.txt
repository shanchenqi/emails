oss-security@...ts.openwall.com Subject: X.Org server security advisory: August 25,   Multiple input validation failures in X server extensions =========================================================  All theses issuses  can lead to local privileges elevation on systems where the X server is running privileged.  * / ZDI CAN XkbSetNames Out-Of-Bounds Access  The handler for the XkbSetNames request does not validate the request length before accessing its contents.  * / ZDI CAN XIChangeHierarchy Integer Underflow  An integer underflow exists in the handler for the XIChangeHierarchy request.  * / ZDI CAN XkbSelectEvents Integer Underflow   An integer underflow exist in the handler for the XkbSelectEvents request.  * / ZDI CAN XRecordRegisterClients Integer Underflow   An integer underflow exist in the handler for the CreateRegister request of the X record extension.  Patches -------  Patches for this issues have been commited to the xorg server git repository. xorg-server will be released shortly and will include these patches.  https://gitlab.freedesktop.org/xorg/xserver.git  commit      Correct bounds checking in XkbSetNames()      / ZDI  commit      Fix XIChangeHierarchy() integer underflow      /  commit      Fix XkbSelectEvents() integer underflow      ZDI-CAN  commit      Fix XRecordRegisterClients() Integer underflow       Thanks ======  These vulnerabilities have beend discovered by Jan-Niklas Sohn working with Trend Micro Zero Day Initiative.   --  Matthieu Herrb  Powered by blists - more mailing lists  Please check out the  Open Source Software Security Wiki, which is counterpart to this mailing list.
