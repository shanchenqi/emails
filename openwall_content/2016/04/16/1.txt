Flaws Fixed in version in march  Changelog is:  * Requests with multiple Content-Length headers will now fail.  * Stop recognizing a single CR (r) as a HTTP line separator. This opened up a possible cache poisoning attack in stacked installations where sslterminator/varnish/backend had different CR handling.   Combinations of theses two flaws in HTTP protocol handling allows for "HTTP Response Splitting" attacks when another actor in front of Varnish3 can transmit headers in this form (for example):      Dummy: header\rContent-Length: 0\r\n  This is a one year old issue, on the old last release of this serie. But we still find some installations.
