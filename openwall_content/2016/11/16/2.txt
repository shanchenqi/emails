The vulnerability could be easily exploited by attackers who have managed to compromise a web application hosted on Nginx server and gained access to www-data account to escalate their privileges to root without any admin interaction thanks to cron.daily.  Vulnerability fixed in the following packages: Nginx package on Debian Nginx on Ubuntu LTS)   The up-to-date advisory can be found at:   For updates, follow:  https://twitter.com/dawid_golunski  Copy of the exploit is also attached to this message.
