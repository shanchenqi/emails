When the user  clicks an element from policyId row of the list, there is a call made underneath  with eventTime parameter which contains the vulnerability. Admin users can  send some arbitrary sql code to be executed along with eventTime parameter  using /service/plugins/policies/eventTime url. Fix details: Replaced native queries with JPA named queries Mitigation: Users should upgrade to version of Apache Ranger with the fix.
