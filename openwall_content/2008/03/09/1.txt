The main problem is that Dovecot's internal protocols use TAB character as a delimiter, but passwords were sent unescaped through them. So passwords containing TAB characters allowed to add new internal fields. The main problem here is a new "skip_password_check" field added in to fix problems with master user logins. Specifying this field allowed the user to skip the password check, as the name implies.
