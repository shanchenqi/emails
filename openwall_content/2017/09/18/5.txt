There are two issues with this functionality (when using SecurityAwareZkACLProvider type of ACL provider e.g. SaslZkACLProvider),  Firstly, access to the security configuration can be leaked to users other than the solr super user. Secondly, malicious users can exploit this leaked configuration for privilege escalation to further expose/modify private data and/or disrupt operations in the Solr cluster.  The vulnerability is fixed from Solr onwards.  Mitigation: 6.x users should upgrade to  Credit: This issue was discovered by Hrishikesh Gadre of Cloudera Inc.
