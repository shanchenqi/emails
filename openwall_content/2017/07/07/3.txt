Irssi Security Advisory [1] ============================================  Description -----------  Two vulnerabilities have been located in Irssi.  (a) When receiving messages with invalid time stamps, Irssi would try     to dereference a NULL pointer. Found by Brian 'geeknik' Carpenter     of Geeknik Labs.      [2] was assigned to this bug  (b) While updating the internal nick list, Irssi may incorrectly use     the GHashTable interface and free the nick while updating it. This     will then result in use-after-free conditions on each access of     the hash table. Found by Brian 'geeknik' Carpenter of Geeknik     Labs. caused by      [3] was assigned to this bug   Impact ------  (a) May result in denial of service (remote crash).  (b) Undefined behaviour.   Affected versions -----------------  All Irssi versions that we observed.   Fixed in --------  Irssi   Recommended action ------------------  Upgrade to Irssi Irssi is a maintenance release in the 1.0 series, without any new features.  After installing the updated packages, one can issue the /upgrade command to load the new binary. TLS connections will require /reconnect.   Mitigating facts ----------------  (a) requires control over the ircd  (b) should not happen with a conforming ircd   Patch -----    References ----------  [1] [2] [3]  Powered by blists - more mailing lists  Please check out the  Open Source Software Security Wiki, which is counterpart to this mailing list.
