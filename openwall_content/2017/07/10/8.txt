The port in question was used by the StatestoreSubscriber class which did not use the appropriate secure Thrift transport when TLS was turned on. It was therefore possible for an adversary, with access to the network, to eavesdrop on the packets going to and coming from that port and view the data in plaintext.   Mitigation:  Users of the affected versions should apply the following mitigation:   - Upgrade to Apache Impala (incubating)   Credit: This issue was identified and reported responsibly by the Cloudera security team.
