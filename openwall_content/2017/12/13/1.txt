A hacker/user can inject/draft the 'sqlSearch' query parameter in such a way to to read/update the data for which he doesn't have authorization.  Mitigation: All users should migrate to Apache Fineract version   Example: A request to retrieve the Clients with displayName=Thomas GET https://DomainName/api/v1/clients?sqlSearch=displayName='Thomas' An attacker/user can use GET https://DomainName/api/v1/clients?sqlSearch= or (1==1) to retrieve all clients in the system  Credit: This issue was discovered by Alex Ivanov  References: http://fineract.apache.org/ https://cwiki.apache.org/confluence/display/FINERACT/Apache+Fineract+Security+Report  Regards, Apache Fineract Team   Powered by blists - more mailing lists  Please check out the  Open Source Software Security Wiki, which is counterpart to this mailing list.
