Thanks  to OSS members for their cooperation.  ======================================================================= Hidden information access issue  Topic:             Data protection issue / Information disclosure by                     "Settings" ->  "Users" ->  "Enrolled users" Severity/Risk:     Minor Versions affected: 2.2 to 2.1 to Reported by:       Andreas Grupp Issue no.:         CVE Identifier:    Changes (master):  Description: Teachers without appropriate permissions were able see user access information.  ======================================================================= Personal communication access issue  Topic:             "Recent conversations" allows anyone to see anyone                     else's messages Severity/Risk:     Serious Versions affected: 2.2 to 2.1 to Reported by:       Juan Aburto Issue no.:         CVE Identifier:    Changes (master):  Description: By manipulating URL parameters, users were able to see others' messages.  ======================================================================= Quiz capability issue  Topic:             When you add a question to the quiz, it does not                     check the question:use... capability. Severity/Risk:     Minor Versions affected: 2.2 to 2.1 to Reported by:       Tim Hunt Issue no.:         CVE Identifier:    Changes (master):  Description: Capabilities were not being correctly checked when adding questions to a quiz.  ======================================================================= Question bank capability issues  Topic:             Various problems with permissions checks in the                     question bank Severity/Risk:     Minor Versions affected: 2.2 to 2.1 to Reported by:       Tim Hunt Issue no.:         CVE Identifier:    Changes (master):  Description: Capabilities were not being correctly checked when working in the question bank. Question authorship was not being checked. Users were shown UI elements when they did not have permission to use them. User permissions were not correctly checked when saving a question.  ======================================================================= Insecure authentication issue  Topic:             CAS Multi-Authentication Does Not Use HTTPS Login Severity/Risk:     Minor Versions affected: 2.2 to 2.1 to Reported by:       Chris Follin Workaround:        Avoid CAS authentication Issue no.:         CVE Identifier:    Changes (master):  Description: A page in the CAS Authentication process was using an insecure HTTP URL that, apart from being insecure, sent the user in circles.  ======================================================================= Information editing access issue  Topic:             Students can edit database entries in read only mode Severity/Risk:     Minor Versions affected: 2.2 to 2.1 to 2.0 to Reported by:       Amanda Doughty Issue no.:         CVE Identifier:    Changes (master):  Description: Students were able to edit pre-existing Database activity entries after the activity had entered a read-only period.  ======================================================================= Capability manipulation issue  Topic:             Non-editor teacher can exceed teacher permissions: example, backup:userinfo Severity/Risk:     Serious Versions affected: 2.2 to 2.1 to 2.0 to Reported by:       Jozas Nhial Issue no.:         CVE Identifier:    Changes (master):  Description: Non-editing teachers were able to redefine their capabilities to achieve actions they would not normally be able to achieve.  ======================================================================= Cross-site scripting vulnerability in Wiki  Topic:             Injection and XSS vulnerability in wiki through                     insufficient validation Severity/Risk:     Serious Versions affected: 2.2 to 2.1 to 2.0 to Reported by:       Sam Hemelryk Issue no.:         CVE Identifier:    Changes (master):  Description: It was possible to inject unfiltered HTML into a wiki page title.  ======================================================================= Cross-site scripting vulnerability in Web services  Topic:             XSS in /admin/webservice/service.php Severity/Risk:     Serious Versions affected: 2.2 to 2.1 to 2.0 to Reported by:       Dan Poltawski Workaround:        Avoid Web services Issue no.:         CVE Identifier:    Changes (master):  Description: The name parameter, sent to the Web service script service.php, was not being filtered correctly.  ======================================================================= Cross-site scripting vulnerability in Blog  Topic:             XSS bug in blog/index.php in IE Severity/Risk:     Serious Versions affected: 1.9 to Reported by:       Simon Coggins Issue no.:         CVE Identifier:    Changes (1.9):     Description: Parameters sent to the Blog module were not sufficiently filtered. This allowed the potential for cross-site scripting in IE  ======================================================================= Potential SQL injection issue  Topic:             Stored SQL Injection in calendar Severity/Risk:     Serious Versions affected: 1.9 to Reported by:       Simon Coggins Issue no.:         CVE Identifier:    Changes (1.9):     Description: It was possible to include unfiltered information when adding a calendar event that was stored in the database.  ======================================================================= Cross-site scripting vulnerability in "download all"  Topic:             Content-Type is TEXT/HTML for zip Download instead                     of application/x-zip-compressed or forced download Severity/Risk:     Minor Versions affected: 2.2 to 2.1 to 2.0 to Reported by:       Asaf Ohaion Workaround:        Avoid "download all" feature in Assignment Issue no.:         CVE Identifier:    Changes (master):  Description: An incorrect mimetype was being reported for zipped assignment submissions, causing some browsers to render the response. The fix for this issue also prevents incorrect use of file sending functions by third-party modules.  ======================================================================= Cross-site scripting vulnerability in category identifier  Topic:             XSS in /cohort/edit.php (POST parameter: idnumber) Severity/Risk:     Serious Versions affected: 2.2 to 2.1 to 2.0 to Reported by:       Dan Poltawski Issue no.:         CVE Identifier:    Changes (master):  Description: The idnumber field, an arbitrary unique identifier for a category, was able to be entered without being filtered.  ======================================================================= Write access issue in Database activity module  Topic:             It's possible for any user to overwrite site wide                     database presets Severity/Risk:     Minor Versions affected: 2.2 to 2.1 to Reported by:       Dan Poltawski Issue no.:         CVE Identifier:    Changes (master):  Description: Users were able to overwrite site-wide Database activity presets created by other users.  ======================================================================= Calendar event write permission issue  Topic:             Calendar New Entry still shows and works for roles                     preventing calendar entry Severity/Risk:     Minor Versions affected: 2.2 to 2.1 to 2.0 to 1.9 to                     Reported by:       Martin Huntley Issue no.:         CVE Identifier:    Changes (master):  Description: Users without appropriate permissions were able to access the new calendar entry page and create a calendar entry.
