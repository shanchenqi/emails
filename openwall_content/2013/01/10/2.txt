A stack-based buffer overflow was reported [1],[2] in Ettercap <=  A boundary error within the scan_load_hosts() function (in src/ec_scan.c), when parsing entries from a hosts list, could be exploited to cause a stack-based buffer overflow via an overly long entry.  In order to exploit this, a user must be tricked into loading a malicious host file.  This has not yet been corrected upstream, but a proposed patch is available [3].
